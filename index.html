1 of 3 Matrix Multiplication — 512 × 512

Imagine multiplying two large spreadsheets cell by cell: each result cell is the sum of an entire row times an entire column. Here we do that for a 512×512 grid — roughly 134 million multiply-and-add operations.

Both sides use a cache-friendly i→k→j loop over Float64Array / Vec<f64>. rustc -O3 auto-vectorises the inner loop using SIMD; V8 cannot guarantee this.

2 of 3 Mandelbrot Set — 800 × 600 px

The Mandelbrot set is a famous fractal image. To generate it we look at each of the 480,000 pixels on screen and repeat a simple formula up to 300 times, checking at each step whether the number has "escaped". No shortcuts — every pixel gets the full treatment.

Purely branchy f64 arithmetic: z = z² + c, bail when |z| > 2. No transcendental functions. Represents visual/scientific workloads with unpredictable branch patterns.

3 of 3 Sieve of Eratosthenes — primes up to 5,000,000

We find every prime number up to 5 million using a classic algorithm: start with a list of 5 million entries and cross out every multiple of 2, then every multiple of 3, and so on. Whatever is left uncrossed is prime. This touches almost every byte in a large block of memory — sequentially, millions of times.

Uint8Array / Vec<bool>, standard sieve. Tests dense sequential memory writes — WASM's linear memory has no GC overhead; the JS engine must track object lifetimes alongside the work.
